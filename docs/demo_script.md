# 演示脚本（5 分钟跑通）

> 目标：当面或录屏演示，5 分钟内展示「JD 解析 → 简历评分 → 排序 → 报告 →
> 面试提纲 → AI 模拟面试」全流程。**零配置即可演示**（默认离线 Mock 模式）。

## 0. 启动

```bash
pip install -r requirements.txt
streamlit run web/streamlit_app.py
```

浏览器自动打开。左侧能看到「离线 Mock 模式」提示（说明无需 API Key 也能跑）。

## 1. 首页 Dashboard（30 秒）

- 介绍系统定位：HR 提效，AI 辅助不决策。
- 指出统计卡片：JD 数 / 简历数 / 筛选数 / 面试数。

## 2. JD 管理（30 秒）

- 进入「② JD 管理」，文本框已预填 `data/sample_jd.md`（AI 应用工程师）。
- 点「🔍 解析 JD」→ 展示结构化结果：must-have、硬技能、关键词被自动拆出。

## 3. 简历筛选（90 秒）— 全程亮点

- 进入「③ 简历筛选」，选刚才的岗位。
- 粘贴 `data/sample_resume_1.md`（张伟，强）到文本框，点「🚀 解析并评分」。
- 再粘贴 `data/sample_resume_2.md`（李娜，弱），再次评分。
- 看**候选人排序表**：张伟 ~80 分「建议面试」，李娜 ~35 分「暂不推荐」。
- 选张伟，点「生成初筛报告」→ 展示**分维度评分 + 证据 + 风险 + 建议面试问题**。
- 强调：每个分都有证据、可追溯；结尾写「仅供 HR 辅助参考，最终由人工确认」。
- 点「⬇️ 导出 Excel」展示一键导出。

## 4. 面试官助手（60 秒）

- 进入「④ 面试官助手」，选岗位 + 候选人张伟，轮次「技术面」，时长 30 分钟。
- 点「🧩 生成面试提纲」→ 展开任意一题：问题 + 为何要问 + 好/差信号 + 追问 + 评分指引。
- 强调：问题对应 JD/简历证据，专挖「看起来强但证据不足」之处。

## 5. AI 模拟面试（90 秒）

- 进入「⑤ AI 模拟面试」，JD 已预填，点「▶️ 开始模拟面试」。
- AI **一次只问一个问题**；随便答一题，提交。
- 展示：AI 先给本题得分 + 亮点 + 不足 + 追问，再进入下一题。
- 故意答一句「不知道」，展示低分 + 引导追问。
- 点「结束并生成报告」→ 展示「JD 模拟面试报告」：总分、分维度、每题复盘、
  能力短板、7 天提升计划。点「⬇️ 下载面试报告」。

## 6. 案例沉淀（30 秒）

- 进入「⑥ 案例沉淀」，展示 `docs/case_study.md`：背景、痛点、方案、表设计、
  可量化指标（初筛 5-10 分钟 → 1-2 分钟）。

## 收尾话术

> 这套系统把「凭感觉」的初筛，变成有证据、可追溯、可复盘的结构化评估；
> HR 从读每一份简历，变成只复核高风险和高潜候选人。
> 架构上预留了飞书多维表格 / Coze / 企业微信接口，可平滑接入企业现有工具链。
