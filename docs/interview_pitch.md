## 一、一句话定位

## 二、30 秒电梯演讲

> 招聘初筛痛点是"凭感觉、标准不一、说不清理由"。我做了一个 HR 提效智能体：
> 输入 JD 和简历，AI 自动抽取信息、按结构化规则**带证据地**打分、生成筛选结论和
> 面试提纲，还能作为面试官对候选人模拟面试。它把单份初筛从 5-10 分钟压到 1-2 分钟，
> 并且每个分都可追溯。技术上用 FastAPI + Streamlit + SQLite，大模型走 OpenAI 兼容接口，
> 还做了离线降级、岗位自适应评分和批量并发。

## 三、3 分钟讲解结构（推荐顺序）

1. **痛点（20s）**：初筛慢、标准模糊、无法复盘、面试提纲拍脑袋、数据不沉淀。
2. **方案（30s）**：双版本——V1 简历筛选，V2 面试智能体；数据库是核心。
3. **现场演示（60s）**：解析 JD → 上传两份简历 → 看排序（强 90+/弱 30+）→
   分维度证据 → 一键导出 → 跑一轮 AI 模拟面试。
4. **技术亮点（40s）**：任务拆分、带证据评分、离线/真实双模、岗位自适应规则、并发提速、熔断降级。
5. **成果与边界（30s）**：可量化提效 + "AI 只辅助不决策、反歧视红线"。

## 四、设计亮点（每条都能展开讲 why）

1. **带证据、可追溯的评分**：每个维度输出 `得分/证据/风险/理由`，不是只给一个分。
   *Why：* HR 要的是"为什么"，可解释才可信、可复盘、可申诉。
2. **任务拆分**（借鉴真实 Coze/飞书 案例经验）：抽取、评分、结论各自独立节点。
   *Why：* "一个模型只做一件事"跑得最快最稳；故障可定位、可单测、可替换。
3. **离线 Mock / 真实大模型 双模自动切换**：没 Key 也能跑完整条流水线。
   *Why：* 零配置可演示（面试现场不怕没网没 Key）；服务层对二者无感知。
4. **岗位自适应评分规则 + 透明化**：按岗位类别模糊匹配规则（任意"…产品经理"用 PM 维度），
   并在界面显示"本次用哪套规则"。
   *Why：* 用工程师的尺子量产品经理必然失真；透明化避免误用。
5. **稳健性**：API 失败（余额/鉴权/超时）自动降级离线 + 熔断；JSON 三重兜底解析。
   *Why：* 大模型输出不可控、外部依赖会挂，生产系统不能因此崩。
6. **性能**：批量简历并发（10 份 4 分钟→30 秒）+ 评分与结论合并调用（单份 ~15s）。
   *Why：* 串行调云端 LLM 体验差，并发是性价比最高的提速。

## 五、关键技术决策与权衡

| 决策 | 选择 | 权衡 |
| --- | --- | --- |
| 评分模型 | 加权指标匹配（命中权重和÷小类最高权重×小类分） | 可解释 vs 端到端黑盒 |
| 总分计算 | 维度分本地求和（不全靠大模型） | 一致可追溯 vs 灵活 |
| 输入格式 | 干净分段文本而非字典原文 | 精度↑、token↓ vs 实现成本 |
| 是否合并抽取与评分 | **不合并** | 守住"一模型一任务"，保抽取/评分质量 |
| PDF 中文 | reportlab 内置 CID 字体 | 跨平台无需外部字体 vs 排版自由度 |

## 六、可量化成果

- 单份简历初筛：5-10 分钟 → 1-2 分钟。
- 面试提纲生成：~15 分钟 → ~1 分钟。
- 批量 10 份评分：~4 分钟 → ~30 秒（并发）。
- 评分从"无依据"到"每维度可追溯证据"。
- 候选人数据从散表 → 结构化数据库沉淀。

## 七、面试高频问题 + 参考答案

**Q：怎么保证 AI 不乱打分 / 不歧视？**
A：分数本地按维度求和、可追溯；prompt 设了校准锚点和反歧视红线（性别/婚育/健康等
非岗位因素不得负面判断，年龄学历仅 JD 明确要求时计权）；低置信标"需人工复核"；
AI 不做最终录用决定。

**Q：大模型返回的 JSON 格式不稳定怎么办？**
A：三重兜底解析（原生→去 markdown 外壳→截花括号+转义裸换行），再用 Pydantic 校验，
失败给兜底结构，绝不让流水线崩。这是从真实案例踩坑里学来的。

**Q：为什么不把抽取和评分合成一次调用更省钱？**
A：试过思路但放弃——"让一个模型同时干两件事"会显著降质、变慢且难排错。
我宁可多一次调用换稳定，并用并发把总时延压下来。

**Q：没有 API Key 怎么演示？**
A：内置离线 Mock（关键词启发式）跑通全流程；配置 Key 自动切真实模型，服务层无感知。

**Q：如何落地到企业？**
A：架构预留了飞书多维表格 / Coze / 企业微信接口；数据库即核心，评分规则可按岗位
用 JSON 扩展，零代码新增岗位。
