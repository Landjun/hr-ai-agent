# 项目沉淀与复盘（HR 提效智能体）

> 这份文档把项目"从 0 到 1 再到打磨"的真实历程、关键决策、踩过的坑与解决方式沉淀下来，
> 既是工程复盘，也是面试时讲"过程与思考"的素材（面试官更想听过程，不只是结果）。

## 1. 一句话

从 0 到 1 独立设计并实现的「简历筛选 + AI 面试」双智能体系统，
**AI 负责提速与给证据，最终决定权始终在人**。

## 2. 迭代历程（按真实开发阶段）

1. **搭骨架**：FastAPI + Streamlit + SQLite + SQLModel，7 维带证据评分，离线 Mock 双模、双智能体（筛选 / 面试），10 个测试。
2. **岗位自适应**：从「医疗AI产品经理」一套专属规则起步，做到「丢一个 JSON 就新增一个岗位」，最终内置 **20 套**（安全 / AI / 数据 / 研发 / 解决方案）。
3. **稳健性**：真实大模型失败自动降级离线 + 熔断；JSON 三重兜底解析。
4. **性能**：批量简历并发（10 份 4 分钟 → ~30 秒）+ 评分与结论合并调用（单份 ~25s → ~15s）。
5. **导出**：Markdown / HTML / PDF / Word + 一键打包 ZIP（PDF 用 reportlab 内置 CJK 字体，跨平台中文不乱码）。
6. **体验与精准**：UI 主题化 + 分维度进度条 + 等级徽章；给大模型干净结构化输入 + 校准锚点，精度↑、token↓。
7. **可配置**：评分规则可在界面查看/编辑权重；JD 一键选用内置示例。
8. **工程化**：ruff lint + CI（双 Python 版本）+ pyproject + 中英 README + 架构图 + 示例产出 + 讲解稿。

## 3. 关键技术决策与权衡

| 决策 | 选择 | 权衡 |
| --- | --- | --- |
| 评分模型 | 加权指标匹配，分数本地求和 | 可解释/可追溯 vs 端到端黑盒 |
| 抽取 vs 评分 | **不合并**（一模型一任务） | 守住质量 vs 省一次调用 |
| 输入格式 | 干净分段文本而非字典原文 | 精度↑、token↓ |
| 岗位规则匹配 | 关键词模糊路由 + 取最贴近 | 易用 vs 偶发歧义（已用"最短标题"消歧） |
| PDF 中文 | reportlab 内置 CID 字体 | 跨平台无外部字体 vs 排版自由度 |
| 大模型失败 | 自动降级 + 熔断 | 永不崩 vs 静默降级需提示 |

## 4. 踩过的坑与解决（最有价值的部分）

1. **「改了 Key 还是离线模式」** → 进程启动时只读一次配置，旧进程有缓存。
   **解决 / 教训**：配置变更必须重启；并在 UI 显式展示当前大模型模式。
2. **「打分很垃圾」** → 真因是**把产品经理简历套到了 AI 工程师 JD** 上（下拉框没切），
   且专属规则按岗位名精确匹配没生效。
   **解决**：改为**按岗位类别模糊匹配**（任意「…产品经理」用 PM 维度）+ 在界面**透明展示**当前用哪套规则、各维度满分。教训：**"分低"常常不是模型错，是输入/配置错；要把判断依据暴露给用户**。
3. **「402 余额不足把页面搞崩」** → 未捕获的 API 异常冒泡。
   **解决**：任何 API 错误自动降级离线；余额/鉴权这类"治不好"的错触发**熔断**，本会话后续直接离线，避免每次白等失败往返（否则明显变慢）。
4. **「还那么慢」** → 串行调云端 LLM 是固有成本。
   **解决**：批量**并发** + 评分结论**合并调用** + 导出结果缓存。
5. **带水印的真实 PDF 抽取出乱码碎片**（姓名被包成 `_ _ 陈志国 _ _`）。
   **解决**：水印噪声行过滤 + 姓名兜底提取（去非中文后取 2-4 字）。

## 5. 可量化成果

- 单份简历初筛：5-10 分钟 → 1-2 分钟。

- 评分从"无依据" → 每维度可追溯证据。
- 20 套岗位规则 / 4 种报告格式 / 25 个测试 / CI 双版本绿。
