# HR 提效智能体案例

> 一个面向 HR / 招聘负责人 / 面试官的 AI 提效系统：自动读 JD、读简历、提取信息、
> 结构化评分、生成筛选结论与面试提纲，并能作为 AI 面试官对你进行模拟面试。
> 核心理念：**AI 只做辅助判断，不做最终录用决定；每个判断都要有证据、可追溯。**

## 1. 项目背景

企业招聘中，HR 每天要处理大量简历，痛点集中在：

- 简历格式五花八门（PDF / Word / 在线简历截图），人工录入慢。
- 初筛标准模糊、因人而异，不同 HR 打分差异大。
- 「凭感觉」筛人，淘汰或通过都说不清理由，无法复盘。
- 面试提纲临时拍脑袋，问题泛泛，问不到关键点。
- 候选人数据散落在邮箱 / Excel，无法沉淀为可复用的人才库。

本项目借鉴了一个真实的「飞书 + Coze 多维表格 HR 工作流」案例的设计思想，把它
落地为一套**本地可运行、可演示、可二次开发**的 Python 系统。

## 2. 原始 HR 流程痛点

| 环节 | 传统做法 | 问题 |
| --- | --- | --- |
| 简历录入 | 人工阅读、手动填表 | 单份 5-10 分钟，易错漏 |
| 初筛打分 | 凭经验估一个印象分 | 标准不一、无依据、不可复盘 |
| 候选人排序 | 手动 Excel 排序 | 维度不统一、靠记忆 |
| 面试准备 | 临场想问题 | 问不到要害、深度不够 |
| 数据沉淀 | 邮箱 + 散表 | 无人才库、无法二次利用 |

## 3. AI 提效目标

1. 自动读取 JD 与简历，抽取结构化信息。
2. 按 JD + 结构化评分规则打分，**输出评分过程与证据**，而非只给一个分数。
3. 自动生成筛选结论、风险点、建议面试问题。
4. 自动沉淀候选人数据库，支持批量排序与一键导出。
5. 自动生成面试提纲，辅助面试官判断。
6. 支持「输入 JD，AI 来面试我」的模拟面试，并生成提升报告。

## 4. 第一版：简历筛选智能体

用户操作流程：

```
输入/上传 JD → 上传一份或多份简历 → 自动解析 → 自动抽取结构化信息
→ 按 JD + 规则评分 → 生成筛选结论 → 候选人排序 → 导出 Excel/Markdown/JSON
```

关键能力：

- **JD 解析**：把口语化 JD 拆成 must-have / nice-to-have / 硬技能 / 关键词等字段。
- **简历抽取**：联系方式、学历、技能、工作/项目经历结构化；缺失字段填「无」，不编造。
- **结构化评分**：7 个维度（共 100 分），每个维度输出 `score / evidence / risk / reason`。
- **筛选结论**：总分、推荐等级、优势、风险、缺口、建议面试问题、是否需人工复核。
- **候选人排序 + 报告**：排序表 + 单人初筛 Markdown 报告，支持多格式导出。

## 5. 第二版：协助面试官智能体

在第一版基础上增加两个模式，**复用**第一版的 JD、简历与评分报告：

- **模式 A 协助面试官**：选 JD + 候选人 + 轮次/时长 → 生成结构化面试提纲，
  每个问题带「为何要问 / 好信号 / 差信号 / 追问 / 评分指引」。
- **模式 B AI 模拟面试**：输入 JD（可选简历）→ AI **一次只问一个问题** →
  你回答后先短评再追问/进入下一题 → 结束生成「JD 模拟面试报告 + 7 天提升计划」。

## 7. 数据表设计

数据库是整个系统的核心（沿用参考案例「多维表格即数据库」思想）。7 张表：

`jobs`、`resumes`、`applications`、`scoring_rules`、`screening_records`、
`interview_sessions`、`interview_messages`。详见 [table_design.md](table_design.md)。

## 8. 智能体能力设计

沿用参考案例最重要的工程经验——**任务拆分**：

> 「不要让大模型同时做两个任务。只让它做一个任务时，跑得最快最好。」

因此 JD 解析、简历抽取、逐维度评分、筛选结论、面试提问、回答点评各自独立成节点，
每个节点单一职责、可单测、可替换、可追溯。详见 [workflow_design.md](workflow_design.md)。

## 9. Prompt 设计

每个 prompt 都包含：角色 / 任务 / 输入 / 输出格式 / 约束（不编造、不确定就标记、
必须输出 JSON 或 Markdown）。详见 [prompt_design.md](prompt_design.md)。

## 10. 评分规则设计

采用「加权指标匹配」模型（来自参考案例）：

> 小类得分 =（简历命中指标的权重之和 ÷ 该小类最高权重）× 小类满分
> 总分 = 各大类得分之和

默认 7 维：岗位匹配度 25 / 必备技能 25 / 项目经验 20 / 工作经历 15 /
教育背景 5 / 稳定性与表达 5 / 加分项 5。规则结构化存于 `data/scoring_rules.json`
与 `scoring_rules` 表，可按岗位调整。

## 11. 演示流程

见 [demo_script.md](demo_script.md)。一句话：解析 JD → 上传两份样例简历 →
看排序（强候选 80+，弱候选 30+）→ 下载报告 → 生成面试提纲 → 跑一场 AI 模拟面试。

## 12. 项目价值

- 把「凭感觉」的初筛变成**有证据、可追溯、可复盘**的结构化评估。
- HR 从「读每一份简历」变成「只复核高风险/高潜候选人」。
- 面试官拿到**针对性**提纲，问到要害。
- 候选人数据自动沉淀为人才库，可二次利用。

## 13. 可量化指标

| 指标 | 优化前 | 优化后 |
| --- | --- | --- |
| 单份简历初筛 | 5-10 分钟 | 1-2 分钟 |
| 批量候选人排序 | 手动 | 自动一键 |
| 面试提纲生成 | ~15 分钟 | ~1 分钟 |
| 评分依据 | 无/口头 | 每维度有证据可追溯 |
| 候选人沉淀 | 散表/邮箱 | 结构化数据库 |
| HR 注意力 | 平均分配 | 聚焦高风险/高潜 |

## 14. 后续优化方向

- 接入飞书多维表格 / Coze / 企业微信（架构已预留接口，见 `.env` 与 config）。
- 按岗位定制评分规则与院校/技能词库。
- 真实大模型 A/B 评测，校准 Mock 与真实模型的一致性。
- 增加 JD-简历语义向量匹配，减少关键词漏判。
- 面试录音转写 → 自动评分与纪要。
